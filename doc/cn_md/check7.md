## 3. 网络拓扑

你们两个各贡献一个主机和一个路由器，通过 cs144.keithw.org 中继服务器连接（因为可能有一方在 NAT 后面）。

### 运行步骤

1. **先自己测试**：在没有同学的情况下，用两个窗口分别运行客户端和服务器，确保代码正常工作

2. **选择端口号**：随机选择一个 1024-64000 之间的偶数（不要用 3000，那是示例）

3. **服务器**（假设选择 3000）：
   ```bash
   ./apps/endtoend server cs144.keithw.org 3000
   ```

4. **客户端**（服务器选择 3000，则客户端用 3001）：
   ```bash
   ./apps/endtoend client cs144.keithw.org 3001
   ```

5. **预期输出**：
   - 服务器：`Listening for incoming connection...`
   - 客户端：`Successfully connected to 172.16.0.100:1234.`
   - 服务器：`New connection from 192.168.0.50:57005.`

6. **验证**：
   - 在一个窗口打字，内容应出现在另一个窗口
   - 尝试双向通信
   - 按 `Ctrl-D` 结束连接，验证程序正常退出

7. **调试模式**：添加 `debug` 参数查看所有交换的以太网帧

8. **与同学测试**：双方都使用自己的代码，分别运行服务器和客户端

## 4. 发送文件

生成随机文件：
```bash
dd if=/dev/urandom bs=1M count=1 of=/tmp/big.txt
```

服务器发送：
```bash
./apps/endtoend server cs144.keithw.org 3000 < /tmp/big.txt
```

客户端接收：
```bash
./apps/endtoend client cs144.keithw.org 3001 > /tmp/big-received.txt
```

比较：
```bash
sha256sum /tmp/big.txt /tmp/big-received.txt
```

如果 SHA-256 哈希匹配，文件传输正确。
